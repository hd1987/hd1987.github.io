nvm usage

nvm --help

显示帮助信息

nvm --version

打印出安装的nvm版本

nvm install [-s] <version>

从源代码下载并安装 , [-s]。 如果可用，使用 .nvmrc

nvm uninstall <version>

卸载一个版本

nvm uninstall --lts

使用自动 LTS（长期支持）别名“lts/*”（如果可用）进行卸载。

nvm uninstall --lts=<LTS name>

使用提供的 LTS 行的自动别名（如果可用）进行卸载。

nvm use [--silent] <version>

修改 PATH 以使用 。 如果可用，使用 .nvmrc

nvm exec [--silent] <version> [<command>]

在 上运行 。 如果可用，使用 .nvmrc

nvm run [--silent] <version> [<args>]

在 上以 作为参数运行 node。 如果可用，使用 .nvmrc

nvm current

显示当前激活的Node版本

nvm ls [<version>]

列出已安装的版本，匹配给定的 （如果提供）

nvm ls-remote [<version>]

列出可用于安装的远程版本，匹配给定的 （如果提供）

nvm version <version>

将给定的描述解析为单个本地版本

nvm version-remote <version>

将给定的描述解析为单个远程版本

nvm deactivate

撤消 nvm 对当前 shell 的影响

nvm alias [<pattern>]

显示所有以 开头的别名

nvm alias <name> <version>

设置一个名为 的别名指向

nvm unalias <name>

删除名为 的别名

nvm install-latest-npm

尝试在当前节点版本上升级到最新的工作“npm”

nvm reinstall-packages <version>

将 中包含的全局 npm 包重新安装到当前版本

nvm unload

从 shell 中卸载 nvm

nvm which [current | <version>]

显示已安装节点版本的路径。 如果可用，使用 .nvmrc

nvm cache dir

显示 nvm 缓存目录的路径

nvm cache clear

为 nvm 清空缓存目录
